**原因分析：**

`exec:java` 目标不会启动新 JVM，而是运行在 Maven 自身的 JVM 进程里。`System.out` 在 JVM 启动时就已经用系统默认编码（中文 Windows 是 GBK）初始化了，之后再改系统属性也来不及了。

**解决方案：**

创建了 jvm.config，Maven 启动时会自动读取此文件并传入这三个 JVM 参数：

| 参数 | 作用 |
|------|------|
| `-Dfile.encoding=UTF-8` | 文件 I/O 默认编码 |
| `-Dstdout.encoding=UTF-8` | `System.out` 编码（Java 17+ 专用属性） |
| `-Dstderr.encoding=UTF-8` | `System.err` 编码（Java 17+ 专用属性） |
